Python Program
    **->** physical lines of code    **|** end with a physical *newline* character
        **->** logical lines of code **|** end with a logical **NEWLINE**  token
            **->** tokenised

**physical** newline vs **logical** newline 

sometimes, *physical newlines are ignored*
in order to *combine multiple physical lines*
into a *single logical line of code*
terminated by a logical **NEWLINE** token

Conversion can be **implicit** or **explicit**

---
### Implicit 

- Expressions in:
    - list literals: []
    - tuple literals: ()
    - dictionary literals: {}
    - set literals: {}
    - function arguments / parameters 
   
**supports inline comments**
##### list

```python
[1, 
 2,
 3]

[1, #item 1
 2, #item 2
 3, #item 3
]
```
##### function 

```python
def my_func(a,
			b,
			c):
		print(a, b, c)

my_func(10 #comment
	   ,20, 30)
```

---
### Explicit 

You can break up statements over multiple lines *explicitly*, by using the \ (backslash) character

Multi-Line statements are not implicitly converted to a single logical line.

[[1.4.Conditionals]]

 ```python
❌  if a 
		and b 
		and c:   

✅  if a \
	   and b \
	   and c:
```

Comments **cannot** be part of a statement, not even a multi-line statement.

```python
❌  if a \
       and b \ #comment 
       and c: 
```

### Multi-Line String Literals

Multi-line string literals can be created using triple delimiters (' single or " double)

```python
'''This is 
a multi-line string'''

"""This is
a multi-line string"""
```

Be aware that non-visible characters such as *newlines*, *tabs*, etc. are actually part of the string - basically anything you type.

You can use escaped characters (e.g. ```\n```, ```\t```), using string formatting, etc.

A multi-line string is just a regular string.

Multi-line strings are not comments, although they can be used as such,
especially with special comments called **docstrings.**

---
##### Implicit Line Continuation

```python
a = [1, 2, 3]

a = [1, 2, 
     3, 4, 5]

a = [1 #item, -> putting the comma like that will end up in SyntaxError
     2]

'''Solution:'''
a = [1 #item 1
    ,2]

'''Tuples Example'''
a = (1 	#comment
    ,2 	#comment
    ,4)

'''Dictionary Example'''
a = {'key1': 1 #value for key 1
    ,'key2': 2 #value for key 2 } -> Also to make sure the closing brace is not on the same line as the comment as above example.

'''Function Example'''
def my_func(a, #this is used to indicate...
            b, #coment
            c):
    print(a, b, c)

my_func(10, 20, 30)

my_func(10,
        20,
        30)

my_func(10, #comment
        20, #comment
        30 	#comment
        )
```
##### Explicit Line Continuation

```python
a = 10
b = 20
c = 30

if a > 5 and b > 10 and c > 20:
    print('yes')

'''Indentation doesn't matter in explicit line continuation'''
if a > 5 \
and b > 10 \
and c > 20:
    print('yes')

a = '''this is a string'''

b = '''this
is also a string'''

c = '''this
    is also a sting 
        that is created over mulitple lines with indentation'''

# Just because it is looking good while implementing in a code doesn't mean it will also look good during the output.
items = '''some items:
           	 	1. item 1
          	 	2. item 2'''

# This will not count the indentation while in function it will be treated as a single entity.
def my_func():
    a = '''a multi-line string
    that is indented in the seond line'''

```
