
Let's say if you want to declare a class in Python

```python
class Rectangle:
    def __init__(self, width, height):
        self.width = width
        self.height = height
```

Here, the first arugment in the ```__init__()``` is the object itself. We typically call it **self**. New object that was just created. Where the ```__init__()``` itself is a **constructor**.

```python
r1 = Rectangle(10, 20)
print(r1.width, r1.height)
```

 Now we'll see artibutes that are callable:

```python
class Rectangle:
    def __init__(self, width, height):
        self.width = width
        self.height = height

    def area(self):
        return self.width * self.height

    def perimeter(self):
        return 2 * (self.width + self.height)

r1 = Rectangle(10, 20)
print(r1.area())
print(r1.perimeter())

# if we want to look the string representation of the class:
print(str(r1))
print(hex(id(r1)))
```

Similarly, ```__repr__()``` however the result might not display here as it is generally more visibile in the Python intreperter and Jupyter Notebook.

```python
r1 is not r2
```

The reason these two *objects* of **Rectangle** class is different because they have different **memory address** even though they have the same values passed in their constructor. But if we do:

```python 
r1 == r2
```

So, what we are going to do about it: 

```python
class Rectangle:
    def __init__(self, width, height):
        self.width = width
        self.height = height

    def area(self):
        return self.width * self.height

    def perimeter(self):
        return 2 * (self.width + self.height)

    def __repr__(self):
        return 'Rectangle({0}, {1})'.format(self.width, self.height)

r1 = Rectangle(10, 20)

print(r1)
```
