
What we have seen so far every time we create a variable i.e. ```my_var = 10``` Python is shorting that value somewhere in the memory address *0x1000* (for example sake). Where **my_var** is a pointer to that object.

We can start keeping track:

```
| reference | count | 
| --------- | ----- |
|  0x1000   |   1   |
```

```python 
other_var = my_var
```

Here, we're not taking the value 10 and assigning it to *other_var* we're actually taking the reference of *my_var* and assigning that to other_var, in other ways it is also pointing to the same reference address.

```
| reference | count | 
| --------- | ----- |
|  0x1000   |   2   |
```

So, the counter will go up to 2 now. Let's say *my_var* goes away it falls out of scope or maybe we assign it to a different object memory or maybe we assign to **None** then that reference goes away and reference count goes to 1. Similarly, when *other_var* goes away the reference count will decrement.

---
### Finding the Reference Count 

```python
sys.getrefcount(my_var) '''passing my_var to getrefcount() creates an extra reference!'''
```

```python
ctypes.c_long.from_address(address).value
```

The difference here in second code is we just pass the memory address (an **an integer**), not a reference - does not affect reference count.

---
### Code Example:

```python
import sys

a = [1, 2, 3]

print(id(a))
print(sys.getrefcount(a))
```

```python
import ctypes

def ref_count(address: int):
    return ctypes.c_long.from_address(address).value

print(ref_count(id(a)))

b = a 

print(id(b))

ref_count(id(b))
```

```python
b = None

print(ref_count(id(a)))
```
