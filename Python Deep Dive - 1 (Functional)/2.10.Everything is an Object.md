
Throughout this course, we'll encounter many data types.
- Integers ```int```
- Booleans ```bool```
- Floats ```float```
- Strings ```str```
- Lists ```list```
- Tuples ```tuples```
- Sets ```set```
- Dictionaries ```dict```
- None ```NoneType```

We'll also see other constructs:
- Operators ```+, -, ==, is, ...```
- Functions
- Classes
- Types 

And many more... But there's this one thing common with all these things, is that they are all **objects** (instances of classes)
- Functions ```function```
- Classes ```class``` [not just instances, but the class itself]
- Types ```type```

This means they all have a **memory address!**

```markdown
def my_func():
    ...

my_func -> function (state) | 0x1000

id(my_func) -> 0x1000
```

As a consequence:
- Any object can be **assigned** to a variable **including functions...**
- Any object can be **passed** to a function **including functions...**
- Any object can be **returned** from a function **including functions...**

They are first class citizens just like any other **variable** 

Here, ```my_func``` without the ```()``` is the **name** of the function. If you use the ```my_func()``` then this would **invoke** the function.

### Code Example

```python
a = 10 
print(type(a))

b = int(10)
print(type(b))
```

Classes in Python generally have the **documentation**


```python
c = int()

```

```python
def square(a):
    return a ** 2 

print(type(square))
```

```python
f = square

print(id(f))
print(id(square))

print(f is square)

print(f(2))
```

```python
def cube(a):
    return a ** 3 

def select_function(fn_id):
    if fn_id == 1:
        return square
    else
        return cube

f = select_function(1)
print(f is square)

f = select_function(2)
print(f is cube)
```

```python
print(select_function(2)(3))
```
